Wildcards

The simplest kind of pattern query uses a word or word fragment with a wildcard character. You can use the asterisk (*) and question mark (?) as wildcard characters in queries. The asterisk matches any sequence of characters, and the question mark matches any single character.

For example, the query esc* matches the terms "escarpment", "escape", and so on. The query r?n matches "run" and "ran", but not "ruin".

In Indexing Service, the equal sign (=) indicates that wildcard characters can be used. If no equal sign is used, Indexing Service uses the Contains operator by default.

The following table provides examples of the long- and short-form wildcard queries:

Short form Long form Result
@filename=*.doc {prop name=filename}=*.doc{/prop}

{prop name=filename}{regex}=*.doc{/regex}{/prop}

 Any file created with a word-processing program (these have the .doc extension)
@filename=readme.* {prop name=filename}=readme.*{/prop}

{prop name=filename}{regex}=readme.*{/regex}{/prop}

 readme.file, readme.
@filename=readme.??? {prop name=filename}=readme.???{/prop}

{prop name=filename}{regex}=readme.???{/regex}{/prop}

 readme.doc, readme.txt
